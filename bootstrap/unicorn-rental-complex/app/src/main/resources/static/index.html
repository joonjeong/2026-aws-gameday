Runbook

API Storyline

The page uses the same sequence the backend expects.

  • POST /api/sessions

    Create a session record and receive the HttpOnly session cookie.

  • GET /api/sessions/current

    Resolve the current session from the cookie and display the active user.

  • GET /api/rentals

    Read the inventory snapshot from Postgres and show current availability.

  • POST /api/orders/reserve

    Reserve an available rental for the active session.

  • POST /api/orders/return

    Return the latest reserved order owned by the active session.

  • GET /api/orders

    Audit reservation history for the same session cookie.
